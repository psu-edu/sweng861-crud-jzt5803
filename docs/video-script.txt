---SLIDE 3 — Architecture---

[01:00]

ASSERTION: Next.js 15 collapses frontend and API into one deploy unit.

Four-line proof on the left: one codebase, one Docker image, no CORS, no split
infrastructure. The architecture diagram on the right shows why — the browser,
API routes, database, and DevOps layer all live in a single Next.js app.
Frontend and backend share one process. One npm build produces the entire
production artifact.

---SLIDE 4 — Technology Stack---

[01:40]

ASSERTION: Every technology choice minimizes overhead and maximizes testability.

The "Why" column is the assertion for each row. SQLite and Sequelize: zero
infrastructure needed to start. Open-Meteo API: free, keyless, no secrets to
manage. Jest plus RTL: 320 tests block broken builds from ever reaching Docker.
Prometheus plus Grafana: SLO dashboards with no external cloud dependency.
Every choice has a specific, testable reason.

[CLICK to next slide]


---SLIDE 5 — Core Features---

[02:20]

ASSERTION: Multi-layer defense delivers seven production-grade features.

On the left — auth and security. JWT, NextAuth, and OAuth all resolve through
one getAuthUser call. bcrypt cost 12 deliberately slows brute force. HSTS and
BOLA checks ship on every response.

On the right — data features. Five server-enforced categories, UUID keys,
US locale formatting, a ten-minute weather cache, and an append-only audit trail
written asynchronously so it never slows the API response.

[CLICK to next slide]


---SLIDE 6 — DevOps and Observability---

[03:00]

ASSERTION: Every commit is automatically built, tested, and packaged. No manual
steps.

The pipeline diagram on the left shows two jobs. Job one: npm ci, build, Jest
coverage — no continue-on-error, so any test failure kills the pipeline right
there. Then npm audit. Job two runs only if job one passes: three-stage Docker
build, non-root user nextjs:1001, smoke test against /api/health five times.

On the right — three observability lines: three health endpoints with distinct
roles, structured JSON logs with sensitive keys stripped, and four Prometheus
metrics powering a five-panel Grafana dashboard. SLOs: 99.5% availability,
p95 latency under 500ms.

[CLICK to next slide]


---SLIDE 7 — AI Audit---

[03:50]

ASSERTION: AI-generated code contained three security flaws — all caught by
manual review.

First, where AI helped: scaffolding the CI YAML, the Dockerfile structure, and
the Prometheus config saved significant time. I would use it again for those
tasks.

But every infrastructure file went through a security review. That review found:
hard-coded JWT_SECRET in the YAML — credentials permanently in git history.
continue-on-error true on tests — broken builds passing the quality gate.
No USER directive in the Dockerfile — container running as root.

Three fixes applied. The lesson is in the table: AI optimizes for working code,
not security defaults.

The five lines at the bottom are technical challenges resolved during
development — each one was a root-cause investigation, not a quick fix.

[CLICK to next slide]


---SLIDE 8 — Testing---

[04:45]

ASSERTION: 320 tests across three isolation layers form a CI quality gate.

The table shows the three layers. Unit tests — 196 cases, no external
dependencies, run in milliseconds. Frontend tests — 124 cases, React Testing
Library in jsdom, no browser. Integration tests — real HTTP calls against a live
server, run locally.

The bottom right cell is the gate rule: pipeline FAILS if any test fails.
The Docker build stage never starts until all 320 pass.

[CLICK to next slide]


---SLIDE 9 — Reflections---

[05:25]

ASSERTION: Observability must be designed in — not bolted on.

Left column — three things built from scratch: a zero-dependency Prometheus
module, a unified auth bridge, and two infrastructure bugs caught only via
end-to-end QA.

Middle column — three lessons: log JSON before you need it, AI drafts but
humans review security, and end-to-end testing surfaces what unit tests miss.

Right column — what breaks at scale: SQLite becomes PostgreSQL, the in-memory
rate limiter becomes Redis, and single-instance becomes Kubernetes.
